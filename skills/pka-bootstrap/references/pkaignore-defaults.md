# .pkaignore Defaults

Default noise filter patterns for PKA. Generated by bootstrap if no `.pkaignore` exists. Uses standard `.gitignore` syntax plus `# @pka-section:` for subtree-scoped rules.

---

## Format

Superset of `.gitignore` syntax. PKA adds subtree-scoped sections:

```
# @pka-section: <foldername>
```

Lines following this comment apply only inside `<foldername>/`. Sections end at the next `# @pka-section:` or end of file.

**Why not `[foldername]` syntax:** Standard gitignore parsers treat `[...]` as character class patterns, causing unexpected behavior. The `# @pka-section:` comment approach is invisible to all standard gitignore parsers.

---

## Default Content

```
# PKA noise filter
# Generated by pka-bootstrap. Edit to suit your repo.
# Uses standard .gitignore syntax plus # @pka-section: for scoped rules.

# ─── Always ignored ───────────────────────────────────────────────────────────
.pka/
.git/
~$*
*.pyc
*.tsbuildinfo
.DS_Store
Thumbs.db

# ─── Build artifacts (global) ─────────────────────────────────────────────────
node_modules/
dist/
build/
target/
__pycache__/
_output/
*.map
*.dll
*.cache
*.class

# ─── Knowledge subtree ────────────────────────────────────────────────────────
# @pka-section: knowledge
knowledge/__pycache__/
knowledge/*.pyc

# ─── Project workspaces — shallow index only ──────────────────────────────────
# @pka-section: projects
projects/*/node_modules/
projects/*/dist/
projects/*/build/
projects/*/target/
projects/*/.git/
projects/*/__pycache__/
projects/*/_output/
projects/*/*.pyc
projects/*/*.map
projects/*/*/*

# ─── Transcript files — explicitly NOT ignored ────────────────────────────────
# Transcripts in team-inbox/ are held for pka-meetings, never auto-routed.
# Do not add *.vtt or transcript patterns to the ignore list.
```

---

## Key Decisions

### Knowledge subtree: minimal exclusions
Knowledge folders are deeply indexed. Only Python bytecode is excluded. Everything else — markdown, PDFs, Word docs, images — is visible to the indexer.

### Project workspaces: shallow index
The rule `projects/*/*/*` skips everything beyond one level inside any project. This means:
- `projects/satori/CLAUDE.md` — indexed
- `projects/satori/alignment.md` — indexed
- `projects/satori/src/main.rs` — **not indexed**
- `projects/satori/docs/architecture/overview.md` — **not indexed**

This is intentional. Project internals are the project's concern. PKA indexes just enough to know what each project is about.

### Code project detection
If a project folder contains `Cargo.toml`, `package.json`, `go.mod`, or `pyproject.toml` at its root, index only top-level `.md/.txt/.pdf/.docx`. Skip all subdirectories for indexing. This prevents source code from polluting the knowledge index.

### Transcripts: explicitly NOT ignored
`.vtt`, `.srt`, and transcript-pattern files must NOT appear in `.pkaignore`. They land in `team-inbox/` and are held for `pka-meetings` processing. Adding them to the ignore list would make them invisible to the transcript detection system.

### Email files: optional
The default includes `*.eml` and `*.msg` as global ignores. Users who want email archives indexed can remove these lines.

---

## Customization Guidance

Users should edit `.pkaignore` before running bootstrap if they have known noise patterns. Bootstrap loads the user's version if present; generates the default only if absent.

Common customizations:
- Remove `*.eml` / `*.msg` to index email archives
- Add `*.log` if log files accumulate in knowledge folders
- Adjust the `projects/*/*/*` depth rule if specific projects need deeper indexing
- Add project-specific exclusions under `# @pka-section: projects`
